# Visual Review

This document owns the current visual language, the capture checkpoints that
matter, and the remaining art/readability work.

## Current Visual Identity

- Local brand fonts live in `public/assets/fonts/brand/`.
- The landing page uses a shader-backed atmosphere layer, voxel particles, and a
  branded ascent brief instead of generic demo framing.
- The in-game view prioritizes horizon readability, route guidance, anomaly
  highlights, and gate-state clarity.
- The shoreline camp acts as the stable visual anchor before realm ascent.

## Mandatory Readability Moments

The game must remain legible in three validated moments:

- start: the player understands orientation and the initial climb
- signal: the active anomaly is readable as an objective
- goal: the gate/finish state is visually clear

These moments are already captured in the browser suite and verified through the
visual manifest.

## Visual Quality Bar

- The landing page should read like a product, not a prototype.
- Realm guidance should support movement, not bury the screen in chrome.
- Biome identity should come from silhouettes, color, and prop choices before
  it relies on text.
- Heavy source models should not be loaded just because they exist.

## Remaining Work In This Domain

- Replace more generic anomaly markers with curated runtime models.
- Add stronger biome-specific dressing and landmark silhouettes.
- Add audio and haptic identity so the presentation is not purely visual.
- Create a tracked store/playtest art set: screenshots, trailer frames, icon
  refinements, and splash polish.
- Review mobile HUD density on smaller phones after more gameplay systems land.
